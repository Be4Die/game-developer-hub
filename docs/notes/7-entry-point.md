# Точка входа

Точка входа — момент, когда пользователь осознаёт потребность в продукте и начинает искать решение.

---

## Разработчики игр

### Сценарии осознания потребности

| Сценарий | Описание |
|----------|----------|
| **Прямое приглашение** | Команда платформы выходит на разработчика напрямую — на конференции, через сообщество, по рекомендации. Разработчик узнаёт об условиях и решает попробовать. |
| **Поиск нового опыта** | Разработчик уже публиковался на других площадках, хочет попробовать что-то новое — другую аудиторию, другие условия, другой подход. |
| **Желание близкого контакта** | На крупных площадках разработчик — один из тысяч. Нет персонального общения, сложно достучаться до поддержки. Платформа привлекает возможностью прямого контакта с командой. |
| **Интерес к веб-играм** | Разработчик хочет выйти в нишу веб-игр или уже работает в ней и ищет профильную площадку с подходящей инфраструктурой. |

### Триггер

> «Хочу публиковать и обновлять игру быстро, без лишних посредников, с нормальной поддержкой и понятными условиями.»

---

## Модераторы платформы

### Сценарий осознания потребности

Модератор — сотрудник платформы, поэтому точка входа определяется рабочей необходимостью:

| Сценарий | Описание |
|----------|----------|
| **Появление инструмента** | Платформа внедряет Game Developer Hub. Модератор переходит на новый инструмент, потому что это становится основным рабочим пространством. |
| **Усталость от текущего процесса** | Накопилась боль от множества чатов, потерянных сообщений, отсутствия единой очереди задач. Новый инструмент воспринимается как решение. |

### Триггер

> «Наконец-то всё в одном месте — тикеты, история, коммуникация.»

---

## Вывод

Для разработчика точка входа — момент, когда он ищет альтернативу текущей площадке или получает прямое приглашение. Ключевой мотиватор — близкий контакт с командой и скорость работы. Для модератора точка входа — внедрение нового инструмента, который решает накопившиеся проблемы с разрозненностью процессов.
